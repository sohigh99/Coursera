  • Specification Specification in EnterpriseOne
    • JDE comprises C function and its specification which determines how a user interfaces and how it is represented. This is important aspect of any enterprise solutions with the layered architecture.

  • Listing of Transaction Processing Listing of Transaction Processing in EnterpriseOne
    • The transaction processing is the core of enterprise solutions which manipulate various data. The applications listed here is tightly associated with MBF (Mater Business Function). Do not add any custom procedure, trigger, function when a certain table is to be manipulated in Transaction Processing (TP) boundaries, which results in the deadlock.

  • Master Business Function Master Business Function in EnterpriseOne
    • The Master Business Functions are the most important components of Enterprise Solutions (ERP, SCM, CRM, Commerce and so on). Commonly, the MBF compries Begin Doc to validate/default header information (business entity) and create cache for it, Edit Line to validate/default detail information (items and others) and create cache for it. Optionally, Edit Doc can be implemented either to check integrity between header cache and detail cache or to call additional routine before commit takes place. Lastly, End Doc gets called with Transaction Processing to commit cache created/validated through Begin Doc and Edit Line to commit to database.

  • Transaction Processing Transaction, Transaction Processing, Commit, and Rollback
    • The Transaction Processing is tightly related with MBF where data is handled using cache when it requires higher integrity among tables. For instance, Item Cost, Item Location, General Ledger, and Cardex for Inventory Transaction.

  • Deadlock Deadlock in EnterpriseOne
    • The deadlock can take place when there is any racing condition - two different process/thread compete to get lock information. In code-current environment, common cause of deadlock is custom code including database level add-on. To have deadlock two different processes are accessing same table with Transaction Processing on. The reason for this behavior is that TP boundaries issues blocking from sql> Begin Trans through sql> Commit Trans, or sql> Rollback Trans. So, it is important to have minimum value for JDENETTimeout because this parameter determines how long it will block.

  • JDEDB_COMMIT_MANUAL JDEDB_COMMIT_MANUAL and TP Boundaries
    • JDEDB_COMMIT_AUTO is used to put a certain table outside Transaction Processing when a certain table need accessing more than one time. The example of this configuration can be Item Location File (F41021) and Item Branch File (F4102) to update MRPC (Net Change Flag) to use during MRP running. In case you need to add a certain routine inside End Doc routine which is called with Transcation Processing, make it sure you call Auto Commit not to cause deadlock.

  • Server Map (SVM920) Sever Map - SVM920
    • The tables listed in this document superceds any configuration you made which controls the data source (or, source of data) and OCM mapping. The environment is defined as the path code (the instruction set of codes which is made up of C functions and specification) and OCM (Object Configuration Management) where the latter determines the source and destination of instruction and data.

  • Business Function Naming Conversion Business Function Naming
    • Basic naming of business function in E1. Some of naming follows World Software naming convention which is the origin of highly distruted JD Edwards system.

  • Functions in JAS Functions in JAS
    • Basically there are two types of functions one is business function and another is system function. There are equivalent set of system function in java and most of embedded (the associated function with data dictionary items) and trigger (either at the DD level or Table level) function have its own java code.

  • Memory Violation Part I Memory Violation Part I
    • The corruption of memory is commonly coming when the memory allocated is different from what memory is assigned, or in referening NULL pointer in nature. The outcome can vary depending on the operating system and its compiler. Commonly, you may not get zombie kernel or memory corruption when your code is up to date.

  • Interoperability Quick Reference Guide Interoperability Quick Reference Guide
    • There are a few of ways to talk to the external systems - EDI, Interop, Business Services. Commonly, interop is used to audit the change made against master files or the tables associated with the master business function.

  • Memory Allocation Failure Memory Allocation Failure
    • Generally, the memory allocation failure is complaint from operating system that the assigned memory (when it is not dynamic) is insufficient to handle a certain task. To overcome the overuse of physical memory, the work file (physical tables) is used in batch applications.
