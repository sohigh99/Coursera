Impact of Changing a Bootstrap Table Data Dictionary Item

Purpose of Document

Symptoms

I have changed the Visual Assist form for Data Dictionary (DD) Alias AN8 to point to a custom Search and Select form and followed the procedures in Document 626511.1 Deploying Data Dictionary Changes. However, the visual assist on the Data Dictionary Alias in the application is still displaying the original Search and Select form instead of the custom Search and Select form. Why?

Cause

A custom change was made to a Bootstrap Table Data Dictionary (DD) Item. Data Dictionary Item AN8 is a table column of bootstrap table F0092.

Solution

Bootstrap tables are primarily system type tables that reside in non-Business Data data sources. Since Bootstrap tables are accessed during EnterpriseOne start-up on the Client and Server, the Data Dictionary Item specs of the Bootstrap tables as well as the table index information are stored in the jdekrnl specs. Therefore, if a custom change is made to a Bootstrap Table Data Dictionary Item or a custom index is added to a Bootstrap table, the jdekrnl specs must be updated first because EnterpriseOne looks into the jdekrnl specs for Bootstrap Data Dictionary Items and table index information.

Updating jdekrnl specs affects ALL environments, based on the recommendation that only one Data Dictionary be shared across all environments. Since jdekrnl specs are delivered with the system foundation code in directory system/bin32, the jdekrnl specs must be updated each time a new Service Pack/Tool Release is installed because a new ServicePack/Tools Release delivers a pristine version of jdekrnl specs.

NOTE:  Oracle Support and Development do not support updating jdekrnl specs with custom changes of Bootstrap Data Dictionary items or custom indexes on Bootstrap tables because it is a global update affecting EnterpriseOne. If for any reason the update affects the running of EnterpriseOne, Oracle Support recommends immediate restoration of the pristine version of the jdekrnl specs.

If you must absolutely have the jdekrnl specs updated due to business requirements and fully understand the impact of updating the jdekrnl specs, open a Service Request (SR) with Oracle Support referring to this Document 644929.1 and request the information in Document 647564.1 How to Update jdekrnl Specs in EnterpriseOne.
