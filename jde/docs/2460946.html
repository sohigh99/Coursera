P41500 Unable To Unreconcile Transactions ('Unreconcile' Button Does Nothing)

Goal

Why P41500 (Inventory GL Reconciliation) does not perform any in clicking 'Unreconcile' button? What is possible cause of it how to overcome the issue?
 

Solution

The processing option of P41500|Version affects the behavior of reconcile and unreconcile as below,

  • Settings
      (Tab - WO Document Type)
      #2. 2. Enter Work Order Issue Document Type = II (* where default value is IM for Material Issue *)
  • The output in P41500
      . Ordinary II: General Ledger Section is empty. By right, P41500 should not allow user to click "Reconcile" in this scenario
      . Material Issue (II) with associated WO: it shows General Leger

 

Reconcile and unreconcile maintains the status by updating columns below,

  • F0911.R3 (Reference 3)  
  • F0911.DKC (Check Cleared)
  • F0911.RCND (Reconciled Code)
  • Where F0911.GPF1 (Inventory Flag) = 1.

And,

  • F4111.SVDT (Reconciliation Date)

So, set different the document types between inventory issue for inventory general and material issue for work order not to have the integrity issue.
