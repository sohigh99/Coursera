Purpose

This document is a checklist of items to assist in setting up Employee Self Service and building workfiles.

Scope

This document is intended for EnterpriseOne users who are setting up the Human Resources module for use within their organization.

Plan Grouping - UDC for use when options are not used for the plan level (EE, EE+1, etc.) or if plans need to be grouped together. Non-participating Plan Flag - if you don't want to enroll in a plan (i.e. medical waive).

G08BB411/P05400 Process Flow Setup - Director Process Flow Setup determines what the employee will see. This is the process flow that controls the event. We send some standard flows but they can be modified to accommodate client needs. E-mail address must be record on Address Book.
IV (Interactive Versions)/P05410 Self Service Director Create version for each event and set up Processing Options. This will be used when identifying which Event is to be attached to the menu for Employee's to execute. Example: ZJDE0002 Marriage, has a processing option where Process Flow Event will be the enrollment event code (i.e. MAR).
G08BB411/P05401 Employee Self Service Director Setup Director Text Application is used primarily for modifying text and/or language. User employee profiles can be changed to indicate a language. Director text can be created in another language, however, applications will still be displayed in English. Can create in Word then formatted in HTML (stored in F00165 with a record type of GT05401).
G08BB413/P085550 Employee Self Service Form Options What fields appear on Forms, Employee Self-Service Form Options, refer to employee personal and contact information.
Use grouping codes to hide information from employees.
Display Option/Disable Input is used to set up events for wizards. Evaluate every field to determine what the employee will see, based on events. This is event specific, so must define this for each event. Tables can be copied and pasted for each event. This is passive security.
IV (Interactive Versions)/P085568 Enrollment Statement Options An Enrollment Statement processing option determines if the employer cost displays. Change processing option for each version. Versions created are used in the P085518.
G08BB412/P085515 Benefits Workflow Approval Setup Work Flow Setup: BENAPPRV is the default version. A version can be defined for every event code.
G08BB413/P085518 Benefit Confirmation Statement Option These options define Button text, statement text, send e-mail. Use this to hide the button to mail the Confirmation Statement.
G08BESS327/P085501 Exclude Plans/Categories from Workfile This application allows users to exclude Plans or Categories from Current and/or Available Plan Workfile Builds. Version specific.

Employee Self Service Workfile Builds

Step Program or Menu Description Item Description or Action
1 G08BESS321/R085530 Current Enrollment Workfile Build

If the Current Enrollment Workfile Build (R085537) processing option is populated with a valid version, current and dependent run concurrently. Review results using P085530W.
2 G08BESS323/R085537 Build Current Dependent/Beneficiary Workfile

Review the results using P085536W and P085537W.
3 G08BESS322/R085520 Available Plans Work File Build

The Process Defaulting processing option is used only for open enrollment. The P085510 (event type, category, etc.) information is checked to verify that defaulting is allowed. Current employee enrollment and future enrollment availability is verified. This processing option requires more processing time, so activate it only for open enrollment. The resulting workfile records include employer contributions. Review the results using P085520W.
4 G08BESS324/R054101 Write Personal Information for Employee

The personal employee information workfile is built with the Address Book, Who's Who, and Phone Information based on the Employee Master. The one processing option determines whether the personal information for the employee, dependents/beneficiaries, or both are included in the workfile.
At this point employees can enroll.
