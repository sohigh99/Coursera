Purpose

This document contains some frequently asked questions and answers on Employee Termination Workflow and assumes all Employee Termination Workflow processes have been activated.

Questions and Answers


Question 1: When some employees are terminated from P0801EMP (Employee) the user receives a pop-up that says "Your changes have been submitted for approval and are pending review" but why are some employee's terminated immediately without any notification message sent for approval?

Answer 1: Employee termination workflow will only send a message for approval to Address Number 7500 (message LM05006 comes from Workflow TERMEMP05) if:

1. There is unpaid Time/unprocessed checks/unpaid DBAs/unpaid SVH and (Processing option in P05W01 is setup for that module)
or
2. The employee is locked

If these 2 items are not true and the Termination Date is today's date or prior the employee will be set to terminated.

Question 2: What are all messages sent from Employee Termination workflow?

Answer 2:
TERMABNOT Termination Aborted Notifications
  sends messages to:
    AN8 for Supervisor LM05019
    AN8 7500 LM05020
    AN8 7505 LM05027
    AN8 for originator LM05027
    AN8 9200 LM05021

TERMBEN Termination of Benefits
   checks P05W01 sends messages to:
    if Batch Enrollment AN8 2049 LM05013
    if HIPPA AN8 2049 LM05009
    if FSA AN8 2049 LM05011
    if Fund AN8 2049 LM05010
    if Enroll AN8 2049 LM05012
    if Cobra AN8 2049 LM05008

TERMEMP01 Termination Initialization (no messages sent)

TERMEMP03 Terminate Employee
   sends message to AN8 for originator LM05005

TERMEMP05 Cleanup unpaid time, checks, DBAs
   if unpaid Time/unprocessed checks/unpaid DBAs/unpaid SVH send AN8 7500 message LM05006

TERMEMP06 Write Time Entry SVH
  if any SVH error sends message to AN8 for originator LM05025

TERMEMP08 Get Employee Information (no messages sent)

TERMNTFY01 Termination Notifications
  sends messages to:
    AN8 7500 (HR manager) LM05015
    AN8 9200 (IT manager) LM05016
    AN8 7505 LM05026
    AN8 2049 LM05014

TERMTIME Terminate Time Entry
  if any SVH sends message to AN8 2049 LM05007

TERMUPDATE EE Termination Update (no messages sent)

Question 3: Can Address Numbers used within Employee Termination Workflow messages be changed?

Answer 3: The Address Number can be changed within the Send Message notification within Workflow Modeler.  This would be customization.

Question 4: When a future Termination Date is used when terminating an employee the employee isn't automatically terminated when that date passes.  Why does the P98860 (Process Task Monitor) for Process TERMEMP01 the sub-process TERMEMP03 show a Halted status?  How can this process complete?

Answer 4: The process is correctly set to WAITFORTD (Halted) is at "Wait for termination date before proceeding".  Workflow has a scheduler that needs to be implemented to check and continue a halted process that meet the criteria.
