Special Handling Code 'O' in UDC (41/I) - Stocking Type Code

Purpose of Document

Information Center: Overview of JD Edwards EnterpriseOne Shop Floor Control > Information Center: Using JD Edwards EnterpriseOne Shop Floor Control > Document1127134.1

This document defines a Special Handling Code 'O' in UDC(41|I) - Stocking Type Code.  To handle transaction flow in both distribution and manufacturing, 'Description 02' (F0005.DL01) and 'Special Handling Code' (F0005.SPHD) are defined with a certain code.

Currently there is stocking types '9' as below,

  • Product Code = "41" - Inventory Management
  • User Defined Codes = 'I' - Stocking Type Code
CodesDescription 01Description 02Special
Handling		Hard
Coded
  Phantom M - Hard Coded   N
9 Special Handling = 'O'     Y

 

  • The 1st character of Description 02 (F0005.DL02) describes whether a certain item is a manufactured part or a purchased part with 'M' and 'P' respectively
  • The 1st character of Special Handling Code (F0005.SPHD) determines whether the item is a project part (P) or a process part (R)

 

Special handling code 'O' for Stocking Type code must be understood along with UDC (31/RS) - Replenishment Source, which is defined below:

In running Kanban Master Revisions (P3016), if 'Replenish Source' is '4' - Outside Assembly and Special Handling code of stocking type to which item belongs  is not 'O' then these errors occur:

  • 117D - Invalid source type for this item
  • 116X - Invalid values for source type


If Kanban is used and an outside operation exists, then the Special Handling Code must be populated with 'O'.  If Kanban is not used, the existing value does not need to be changed.


All the major stocking types are explained and described in <Document 791175.1> - E1: 40: Overview of Stocking Types in EnterpriseOne
