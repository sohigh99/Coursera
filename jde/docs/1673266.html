Purpose

 This document outlines the steps required to set up and apply Payment Terms (01) and Paymnet Terms by Branch (38) Preference at Sales Order Entry.

Details

Overview

Payment Terms: P40301-F40301 (Preference Type 01)

The Payment Terms preference is used to default the Payment Terms and/or Payment Instrument into the Sales Order Detail file (F4211).

Payment Terms by Branch/Plant: P40338-F40338 (Preference Type 38)

The Payment Terms by Branch/Plant preference works the same way as the Payment Terms preference type 01, but it adds the flexibility to have the preference apply to only sales order detail records with a specific Branch/Plant defined in the preference.

Set Up

Preference Definition (Menu G4231)

From Preference Master (P40070), inquire on Payment Terms (01 or 38) and take row exit to Hierarchy (P40073).



Verify that the hierarchy is set up as desired.



To set up the preference, in Preference Master (P40070) take the row exit to Profiles, then add the new preference. Alternatively, the preference profiles can be accessed directly in P40301 (Payment Terms Preference Profiles) or P40338 (Payment Terms by Branch Preference Profiles).

Hierarchy selections – in this example, the preference will be set up by Item / Customer.



Enter the preference set up information.

 

Note: Both Payment Terms and Payment Instruments must have a value – neither option can be left blank in the preference setup.


Sales Order Entry (Menu G4211)

Sales Order Entry (P4210) Processing Options:

Preference Tab – activate preference processing



Versions tab – version of Preference Profile (R40400) is the default, which is ZJDE0001



Preference Profile Processing (R40400) – activated Payment Terms

Or Payment Terms by Branch

Example at Sales Order Entry

Enter a sales order for the item and customer defined in the preference.



Take the SOE - Additional row exit to see that the payment terms and the payment instrument have now defaulted based on the setup in the preference.
