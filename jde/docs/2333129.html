P400511 Issue '018A' Error on UDC 42I/OL Against Hidden Column OLTY After Applying Baseline ESU for Sales

Symptoms

Return Material Authorization Revisions (P400511) issues error as below,

not found in User Defined Code
An error has occurred in the grid on a hidden column (that is, a column that you cannot view). Please contact your system administrator.
not found in Use Defined Code 42I OL
Error Details:
    Form                          P400511_W400511E
    Control Id                   1
    Control Title                RMA Grid (Outbound Inventory Location Type)
    Error ID                       018A

 

Even though Inventory Outbound is not in use (SY42I - Use Outbound Inventory Module) in EnterpriseOne System Control (P99410) ( is No (0 or N).


Changes

All baseline ESUs are in place in Sales area for EnterpriseOne Application release 9.1. 

Cause

Error '018A' gets issued when UDC does not have any definition on 42I/OL (Outbound Location Type).
 
Issue can be replicated as described in the <Bug 26956208> - P400511 ISSUES ERROR 018A ON UDC 42I/OL AGAINST HIDDEN COLUMN OLTY

  

Note that above bug is not changing any code. Not to have the error describe here, follow the instruction in Solution section below

 

Solution

emp__

 

Not to have error in P400511,
1. Log on to E1
2. (Fast Path) UDC
3. Enter the Product Code '42I'
4. (Form Exit) Cost Types

Note that similar issue is reported in P4101|W4101C [Additional System Information] when you do not have blank code for UDC 43A/AP, 43A/CA. See <Note 1982628.1> - E1: 40: Troubleshooting Error 018A - Not Found in User Defined Code. Currently, there is no way to correct the hard error and save the transaction because,

  • Issue is caused when a newly added DD item has an embedded Edit Rule with UDC value
  • Columns are hidden
  • The default value does not exist from UDC table so error is coming during validation of DD items in JAS. Currently, there is no means to correct this error other than adding blank code for each UDCs.
