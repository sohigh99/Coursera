Inventory Transaction MBF - XT4111Z1

Purpose of Document

Disclaimer: The audience for this note is someone with developer level knowledge.

In distribution and manufacturing module inventory transaction is crucial and this document is to explain how XT4111Z1 is utilized.

Note:
  • It is not good practice to call this business function directly from outside. Rather let other master business function call equivalent routine:
    • By doing this it is easy to set transaction boundaries for manual commitments
    • Inventory transaction is subset of daily business routines (ship confirm, purchase order receipt and so on)
    • Unlike other master business function XT4111Z1 does not have Begin Doc routine. Most of case Begin Doc will validate header and create cache for header usually customer and supplier. Whereas XT4111Z1 is meant for inventory transaction which is detail information for transaction in the module
  • Validation takes place from most generic to most specific data set
  • It will be working fine in calling XT4111Z1 directly but it may require higher level of knowledge to handle parameter defined properly

When XT4111Z1 is utilized?
Master business function XT4111Z1 is made up of below business functions:

Function Name Description Template Name Others
F4111EditLine F4111 Edit Line DXT4111Z1C Validate detail routine and write it into cache F41UI001
F41UI001ProcessInventoryCache F41UI001 Process Inventory Cache Record DXT4111Z1J This does not belong to general routine
F4111EndDocument F4111 End Document DXT4111Z1E Commit Caches to tables and call F4111ClearDetailStack
F4111ClearDetailStack F4111 Delete Work File DXT4111Z1I Delete cache not only F41UI001 but also all the caches created throughout transaction

Note:
BSFN F41UI001ProcessInventoryCache is not mandatory for inventory transaction routine rather this is used to access cache directly from a certain routines.

A. In accessing cache below 8 value will be used:
  • F41UI001.itjobs- mnInvJobNumber
  • F41UI001.itlnid - mnOrderLineNumber
  • F41UI001.itdoco - mnOrderNumber
  • F41UI001.itctid - szComputerID
  • F41UI001.itdcto - szOrderType
  • F41UI001.itkcoo - szOrderKeyCompany
  • F41UI001.itsfxo - szOrderSuffix
  • F41UI001.itnlin - mnLineNumber (except PO receipt)
B. With cActionCode as below:
  • 1, A or a: Add (Error 3791)
  • 2, C or c: Change/Update
  • 4, I, or i: Inquiry (Fetch One)
  • 5: Multiple Fetch
So cActionCode has to be 2, 4 and 5 (mostly cActionCode will be 4 to inquire exiting cache.

What does XT4111Z1 do?
XT4111Z1 validates input data and update/insert Item Location, Cardex and Cost information depends on the process you are on.

Data Structures of XT4111Z1:

  • F4111EditLine (DXT4111Z1C): This is the function to validate input values so verify output values
  • F4111EndDocument (DXT4111Z1E): This is the function to deal with returned value from edit line
  • F41UI001ProcessInventoryCache (DXT4111Z1J): Since Edit Line has created cache so this function is to take input values
  • F4111ClearDetailStack (DXT4111Z1I): If all the transaction is done properly, this function will delete work files based on returned value from edit line

How related tables are added/updated through End Doc?
Through F4111EditLine inventory cache (F41UI001) and G/L Cache (I09UI002, I09UI003) is created. Through End Doc routine cache will be committed to related tables.

  • Update Average Cost (F4105 - Item Cost): <Document 1523038.1> E1: 41: Case Study - Slow performance in Inventory Transaction (Example of P4116 - Item Reclassification)
  • Update G/L (F0911 - General Ledger)
  • Update Audit Information (F4102 - Item Branch)
  • Update Quantity (F41021 - Item Location)
  • Update Agreement (F38011 - Agreement Quantities)
  • Write Cardex (F4111 - Item Ledger)
  • (Optional EDI) write F47126/F47127 - EDI Product Activity Data Header/Detail
  • Write cost computed (F4105 - Item Cost)
  • Update/Write Item Price (F4106 - Item Price)
  • Write/Update Lot (F4108 - Lot Master)
  • Write License Plate
  • (Optional) Update F4115 - Item History (cUpdateSalesHistory = 1)
  • Update F3106 - W.O. Cross Reference File (P31114 and DFM)
  • Update F4102.MRPC (net change flag)
  • Delete F4141 and Update F4106 (for purge option is ON for R41610)
  • Delete F4141 and Update F4140 (when purge is ON for R41413)
  • Delete F41021WF
  • Call R47127O - Outbound Product Activity Data Subsystem UBE
  • Write Bulk Information (SY41B is installed from F99410)
  • Update Warehouse
  • Close batch by updating F0011.IST
  • Delete all the caches created/used




What is routine to write F4111 (Cardex File)?
Any application listed above can call routine to write F4111 as below,
Application
|--- (Post OK button clicked) xxx End Doc Routine (for example XT4312Z1 - F4312EndDoc)
|--- F4111EndDocument (XT4111Z1 - F4111 End Document: To commit F41UI001 cache to F4111 via below routine)
|--- InventoryCardexShell (B4100050 - F4111 Format Item Ledger Record)
|--- WriteCardexRecord (B4100040 - F4111 Write Item Ledger: this routine inserts data into F4111)
|--- GetNextUniqueKeyID (X00022 - Get Next Unique Key ID)
|--- JDB_InsertTable()

Note: X00022 gets defined F00022.UKID for table F4111 through Locking API JDB_FetchKeyedForUpdate() and update F00022.UKID immediately through JDB_UpdateCurrent () based on *ROWID. So in normal situation duplication of F4111.UKID is very unlikely.
