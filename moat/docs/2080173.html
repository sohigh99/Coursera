Listing of Header Files Containing typedef for Processing Option Template

Purpose of Document

The purpose of this document is to list the objects which have definition of Processing Option Data Structure (namely template). Some of major application has sub-routine which has definition for processing option template data structure. Possible reason for this type of configuration is that it is less expensive than putting it into the header file of master business function and other application/routine can make use of it without referencing master business function if needed.

Commonly, JDE.log may contain message like,

RUN0000066 - Warning - ProcOpt Data Size Mismatch: Requested 1936 is less than Retrieved 1938 for App P4310,Version ZJDE0001.
Data Structure allocated successfully. This usually means items have been added to the PO template and existing business function may function correctly.

And this document lists application, processing option template and business function relationship, which sometimes does not follow naming rule as you expected.


For easier understanding of this document, example of P4310 (Purchase Order Entry) is to be used,

  • Application ID: P4310 (Purchase Orders)
  • Processing Option template: T43101 (Purchase Order). This information can be found either through Form Design Aid (File > Application Properties) or F9860.MID1 column value
  • Related master business function: XT4311Z1/XT4311Z2 (Purchase Order Entry)

Simplified implementation:

F4311FSBeginDoc ()
|--- IXT4311Z1_BeginDocInitializeProcessingOptions()
|--- CALLIBFRET (idReturnValue,IB4301710_GetP4310ProcOpts_XT4311Z1(lpBhvrCom, lpVoid, lpdsProcessingOptions)) ;
|--- lpdsPrOpt = (LPDST43101) AllocatePOVersionData (hUser, lpDS->szProgramId, lpDS->szVersion, sizeof(DST43101)) ;
