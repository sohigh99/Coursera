Distribution and Manufacturing Constant

Purpose of Document

The audience for this note is someone with developer level knowledge.

1154087.1_f4009_f4009t_f4009t1This document is provides simplified mapping on columns from F4009, F4009T and F4009T1 with parameters and controls on form. And this document can be used where the setting reside and what are the settings from F4009.

Note:
  • This note is written based on EnterpriseOne application release 9.0
  • Most of column/flag may not be descriptive
  • Different E1 release may not have a certain columns
  • Currently value from F4009 can be accessed through the Branch/Plant Constants (P41001).
  • F4009 contains only a single record with blank system code so all the values are to be accessed through Form Menu
  • All the columns define from F4009 are to be repeated either in 'System Constants' and 'Pricing Constants'
  • F4009T (Distribution Manufacturing Constants Tag File) is used only in Xe and ERP8
  • In 8.9, 8.10, 8.11, 8.11sp1, 8.12 and 9.0 columns in F4009T are merged into F4009 (Distribution/Manufacturing Constants)
  • A new table F4009T1 (Distribution/Manufacturing Constants Tag Table) is added to hold information on Taxed Sales Price

Note:
  • F4009T1 is available in EnterpriseOne application release 9.0 and above
    • <Document 1329908.1> - April 5, 2011 - JD Edwards EnterpriseOne Sales Order Tax Inclusive Pricing
    • <Document 1234293.1> - JD Edwards EnterpriseOne Taxed Prices for Sales Orders 9.0
  • If Taxed Prices Enhancement is in place, some of routine may replace the call to B4000530(Fetch from F4009) with B4000560 (Fetch from F4009 and the tag table for this enhancement F4009T1).
  • In changing any value from F4009/F4009T1 database cache has to be reset as the table F4009/F4009T1 are a member of table F98613 which is database cache.
  • The information written above may be changed without prior notice to reflect constant enhancement which Oracle is executing.
  • If columns are not defined above those are coming from the Inventory Constants (F41001) table.
  • If a certain columns are written at form level menu, look for data based on system code (for example, Application Constant information can be found WHERE CISY='42').
  • If database cache of F4009 is having problem the File can not be accessed (error 3143) may appear. Possible cause can be either database cache is having problem or for the new environment this table is empty. To resolve this issue:
    • Check data through UTB or Databrowse by allocating Distribution/Manufacturing Constants (F4009) table
    • If there is data then delete database cache
    • If this table (F4009) is empty (and no row security) then:
1. Go to Branch/Plant Constants (P41001)
2. (Form Exit) System Constants and verify the default value
3. Click OK which will create a record into F4009
4. (Fast Path) WSJ - Work with Server Job
5. Highlight the logic server (e.g., MYPEDB1)
6. (Row Exit) Reset Cache
7. Tick "Database" and click OK
