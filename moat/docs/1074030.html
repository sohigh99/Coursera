Item Cost and Price Update using XT4111Z1

Purpose of Document

In EnterpriseOne, some applications which are related with inventory transaction may update Cost and Price information. Unlike writing Average Cost, this type of transaction may take place. So this document is to explain what are the list of applications which may (or may not) write Cost and Price information.

Q2> What is the Business Function which handles Cost and Price Add/Update?

Above routine is handled by XT4111Z1 - F4111EditLine

Q3> Does it add cost information so long as above listed applications are running?

No. It is working conditionally,

  • Condition 1: F4105 - Item Cost does not have data meet Item/Branch/(Location/Lot) (F4105 does not have a record for combination of item/branch (and location/lot) depending on cost level defined in item master)
  • Condition 2: Costing Method Method (F4105.LEDG) is not 07 - Standard Cost
  • Condition 3 - F4101.CLEV (Cost Level is 1 - Item Only or 2 - Item/Branch Only)
F4101.SRCE - Lot Process Type is 'Space' or 'Blank' or NULL
OR (Input Lot Number is Not Blank AND Lot Number is Not Blank)
OR Lot Process Type is 2 - Lots Assigned with Next Number

  • Condition 4 - F4101.CLEV (Cost Level is 3 - Item/Branch/Location/Lo )
        F4101.CLEV (Cost Level) is 3 - Item/Branch/Location/Lot
        AND (F4101.SRCE is Not Blank AND Input Lot Number is Blank)

Only when "Write Cost" is 'Y' and meets above condition(s) then it write a new record into F4105 - Item Cost File

Q4> When does it Write Price (F4106 - Item Base Price File)?

Only when it meets below condition it write a new record into F4106 - Item Base Price File,

  • Condition 1 - In calling F4111EditLine (F4111 Edit Line) Flag F41UI001.ITAPRI has to be '1' but there is no implementation for this yet
  • Condition 2 - F4101.PLEV (Sales Price Level) is 2 - Item/Branch Only OR 3 - Item/Branch/Location/Lot
  • Condition 3 - F4106 does not have Price Information for input Item/Branch/(Location/Lot) combination
  • Condition 4 - Application ID meets above metrics

 

Q5> Then when does it update item cost into F4105 - Item Cost File?

When it meets below conditions,

  • Application ID meets above metrics
  • F4105.LEDG (Costing Method) is "06 - Lot"
  • cUpdateUnitCost Flag in calling F4111EditLine is '1 - Update Cost'

 

Q6> In copying existing application P4112 into P554112 will same functionality retained?

Yes it will retain same functionality as it determined by how BSFN F4111EditLine gets called

 

Q7> What are the list of applications which add/update Average Cost?

List of applications defined in UDC(40|AV) or UDC (40/AV) - Average Cost Calculation. So long as Description 02 (F0005.DL02) is defined 'y' or 'Y' it shall update Average Cost. On Average Cost refer <Document 625564.1> E1: 41: Weighted Average Cost.

Note:
  • Do not add additional application as it may not update average cost a certain application is not calling XT4111Z1 - F4111EditLine. For example, though you add P4210 in the list, which won't update Average  Cost

  • Average Cost will get update only when cUpdateAverageCost paramter is '1' in calling F4111EditLine. So in case there is issue in updating average cost, capture jdedebug.log and look into this flag
