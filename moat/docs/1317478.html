What is Batch Control?

When users create a batch of documents using an interactive application such as Journal Entries,they may want to enter the total number of documents and total amount expected for each batch. Batch Control allows users to verify the total amount expected against the total amount entered immediately after the batch is created. Batch Control is enabled by checking the Batch Control Required check box in the General Accounting Constants. With batch control enabled, as you enter your application the batch control form will display and allow you to enter the expected amounts. As you exit the application, the batch control form will again display and allow you to compare the expected amounts to actual amounts.

N0000001 - BatchOpenBatchControlProcessing (Batch Open - Batch Control Processing)

  • Purpose: Important: This Business Function is a Client-Only Interpretive Named Event Rule(NER) which will be run as Java serialized object code. This business function cannot be run through CallObject Kernel on the server because, business functions running through CallObject Kernel cannot call any Interactive Application. Thus, this business function is set as a Client-Only Interpretive NER and will be run as a Java Serialized object on the web server. This business function will call an interactive application P0011.
  • History:

    The Batch Control Processing functionality has been removed from B0000004 (F0011 Write Batch Header). A new Interpretive Named ER (client-only NER), N0000001 - Batch Open - Batch Control Processing, was created to handle the Batch Control processing. Due to this change, applications and business functions that use Batch Control need to be modified to use N0000001 instead. To minimize the impact of this change, other functionality in B0000004 was kept intact. From this point forward, we recommend to use B0000004 in UBEs and business functions where Batch Control is not needed and use N0000001 in applications or NERs where Batch Control is needed.

    This change will have an impact on applications and business functions that are using Batch Control. For applications, the calls to any Open Batch business functions (i.e. B0000004 and B0000042) should be replaced with the new business function N0000001. Applications that use B0000010 to call batch control directly should be modified to use a Form Interconnect. For business functions, to keep the Batch Control functionality, they should be re-written in NERs.

    In order to minimize the impact of the business function changes, the data structure (D0000004) used by B0000004 is not changed. However a couple of parameters are obsolete from B9 onwards. The data structure (D0000017) used by N0000001 are the same as D0000004. So the parameter mappings can be kept the same when replacing B0000004 with N0000001.

    • And insert it into F0011 - Batch Header File (IMPORTANT: Creation of batch header is done by this NER and to trace it, review jasdebug.log not jdedebug.log)

N0000001 - BatchControlProcessing (Batch Control Processing)

  • Purpose: This Business Function is a Client-Only Interpretive Named Event Rule(NER) which will be run as Java serialized object code. This business function cannot be run through CallObject Kernel on the server because, business functions running through CallObject Kernel cannot call any Interactive Application. Thus, this business function is set as a Client-Only Interpretive NER and will be run as a Java Serialized object on the web server. This business function will call an interactive application P0011. N0000001 - Batch Open - Batch Control Processing gets called in the beginning of transaction whereas this BatchControlProcessing (Batch Control Processing) gets called in the end of transaction in a certain interactive application.
    • Parameters
    • Data Structure: D0000001 - Batch Control Processing

cBatchControlCancelFlag EV01 JCHAR I Depends on application. Mostly this value is not handled. Only when this flag is assigned as 1 it exits from routine (so Batch Control is N)

  • Detail Routine:
    • This NER is to be called after B0000042 - FSCloseBatch (F0011 Close Batch)
    • If Batch is still in processing this NER is calling Call( App:P0011 , Form:W0011B , Version: ZJDE0001 ) to indicate batch is still in processing in a certain interactive application
    • When batch is processed then it calls Call( App:P0011 , Form:W0011F , Version: ZJDE0001 )

B0000008 - BatchRevisonOnExit (F0011 Update on Exit)

  • Purpose: This Business Function will close a batch header record and is designed to be executed when "Cancel" or "Close" button is pressed in either Add or Update Mode.
  • Parameters
    • Data Structure: D0000008 - Batch - Revise On Exit
Parameter Name Data Item Data Type I/O Req Purpose
mnBatchnumber ICU MATH_NUMERIC I
szBatchtype[3] ICUT JCHAR I
cPreviousBatchStatus IST JCHAR I
cFLAGBatchStatus EV02 JCHAR I
  • 1 - set batch status to value passed in
  • 2 - set status to ERROR 'E'
  • Others/Default (refer detail routine)
cFLAGTransactionInBalance EV03 JCHAR I
cFLAGSuppressBatchControl EV04 JCHAR I
cOverrideMode EV01 JCHAR I
cSuppressDeleteFlag EV01 JCHAR I

  • Detail Routine:
    • If the Suppress Delete Flag allows deleting an empty Batch Header Table (F0011) record and the number of document written in batch header is 0 then this batch header can be deleted.
    • If input parameter cFLAGBatchStatus is '2' then IST ='E'
    • If batch is Error and BatchType is "1" and cUseManagementApprovalFlag = 'N' set IAPP (Batch Approved for Posting) = 'A'
    • If batch is Error and BatchType is "1" and cUseManagementApprovalFlag = 'Y' then set IAPP = 'P'
    • If cFLAGBatchStatus is either 1 nor 2
      • cUseManagementApprovalFlag = 'Y' then IST = ' ' (a space) and IAPP = 'P'
      • cUserManagementApprovalFlag = 'N' then IAPP = 'A', IST ='A', DRSP =' ', 52PP = ' ' and CBP = ' '
    • If entered amount is same with computed amount then set BAL = 'Y' otherwise set 'N'
    • cFLAGTransactionInBalance ='1' OR cFLAGTransactionInBalance = 'Y' then set BALJ='Y' otherwise set 'N'
    • Error detail:
Error Code Error Detail Cause
078D Open for table F0011 failed OpenTable F0011 failed
078I Fetch from table F0011 failed JDB_FetchKeyed() is failed against F0011
078G Delete from table F0011 failed JDB_DeleteTable() F0011 failed
078H Update to table F0011 failed UpdateTable F0011 failed
  • In custom code if this is implemented wrongly, there is chance of experiencing locking or deadlocking

B0000007 - BatchReviseOnTransactionCommit (F0011 Update on Transaction Commitment)

  • Purpose: This Business Function will update a batch header record and is designed to be executed when "OK" is pressed in either Add or Update Mode.
  • Parameters
    • Data Structure: D0000007 - Batch - Revise On Transaction Commitment
Parameter Name Data Item Data Type I/O Req Purpose
mnBatchnumber ICU MATH_NUMERIC I/O
szBatchtype[3] ICUT JCHAR I/O
mnTransactionAmount AME MATH_NUMERIC
mnTransactionAmountPrevious AME MATH_NUMERIC
cPreviousBatchStatus EV01 JCHAR
cFLAGBatchStatus EV01 JCHAR
cFLAGTransactionInBalance EV02 JDEDATE
cFLAGSuppressBatchControl EV03 JCHAR
cFLAGFormExitedOnAdd EV04 JCHAR
cOverrideMode EV05 JCHAR Indicate SV Form_Mode or action taken (Valid value: A - Add, C - Change, D - Delete and V - Void)
mnTotalNumberOfDocuments MATH01 MATH_NUMERIC

  • Detail Routine:
    • AME (Amount - Entered) is determined based on input cOverrideMode as below,
      • V - Void: AME = AME + mnTransactionAmount
      • C - Change/Update: AME = AME - mnTransactionAmountPrevious + mnTransactionAmount
      • D - Delete/Cancel: AME = AME - + mnTransactionAmountPrevious
    • DOCN (Total Number of Document) is computed as below,
      • When input mnTotalNumberOfDocuments is not zero and cOverrideMode is 'A' then DOCN = DOCN + mnTotalNumberOfDocuments
      • When input mnTotalNumberOfDocuments is not zero and cOverrideMode is 'D' then DOCN = DOCN - mnTotalNumberOfDocuments
      • When input mnTotalNumberOfDocuments is zero and cOverrideMode is 'A' then DOCN = DOCN + 1
      • When input mnTotalNumberOfDocuments is zero and cOverrideMode is 'D' then DOCN = DOCN - 1
    • If closing the batch while updating it, then set various table column values of the Batch Header Table (F0011)
    • cFLAGFormExitedOnAdd = 1 and DOCN = 0 then delete Batch Header
    • cFlagBatchStatus is same with detail routine in BatchRevisonOnExit
    • Error detail:
Error Code Error Detail Cause
4364 Invalid Parameter cOverrideMode is not in ('A', 'C', 'D', 'V')
078D Open for table F0011 failed JDB_OpenTable() failed
078I Fetch from table F0011 failed JDB_FetchKeyed() Failed
078G Delete from table F0011 failed JDB_DeleteTable() Failed
078H Update to table &1 failed JDB_UpdateTable() Failed

Pre-defined Batch Status
: UDC 98/IC or UDC 98/IC - Batch Status

Product Code Us Cd User Def Code Description Description Special Handling Reference
98 IC Pending Default Value
98 IT A Approved
98 IT D Posted Done
98 IT E Error
98 IT P Pending In Posting
98 IT U In Use To prevent other process handle a specific batch

  • A certain batch can be accessed only when the batch status is A - Approved.
  • Batch Status 'U - In Use' is to be updated as soon as batch is accessed. In this way other processes/users are not able to access the same batch till end of batch process. This can can help to avoid phantom read or dirty read of a certain transaction.
  • Batch Header (F0011) is to be written as soon as monetary transaction is to be handled and this will be used till end of transaction.
  • The batch type and batch number combination makes a set of transaction and determine how Account Ledger is to be handled in writing General Balance file (F0902).
