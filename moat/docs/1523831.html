Grid Collapses in Form W4314B (Receipt To Match) for Language CT

Purpose of Document

The purpose of this document is to guide the user how to debug the grid collapse issue step by step. Note that this is one example and the actual cause of the issue can be different from what it is described here.

Grid collapse (memory corruption in HTML client) can come from,

  • Data impurity (or data corruption)
  • Specification corruption

this document shall focus on specification corruption

Symptoms

When using Chinese traditional (CT) language profile in User Profile (P0092), called form W4312B (Receipt To Match) does not show grid (Grid collapse) in HTML client.

  • (Testing purpose) it is working fine in Windows client (F9860.siansif = ' ' where siobnm in ('P0411', 'P4314')
  • It is working fine in CS (Chinese Simplified)
  • Steps to duplicate:
    • 1. Add a PO via P4310 PO Entry.
    • 2. At the P4312 perform PO Receipt.
    • 3. At the P4314, enter the PO number and branch etc. Form exit to Receipt To Match (P0411|ZJDE0007 is calling P4314 (W4314A)
    • 4. Grid collapse

Cause

Corrupted FDATEXT specification in HTML client.

Note:

  • Grid can be collapsed owing to corrupted data (in general in contains special character)
  • Refer to <Document 947798.1> - E1: FDA: Grid Disappears When Using "Go To End" on Web Client

Workaround

In case this is an urgent issue, correct it through OMW. Note that Form Title can not be overridden through vocabulary override. Refer to- E1: DD: Vocabulary Overrides.

Before starting review central object information through UTB (Univeral Table Browser),

Note:

  • Language specific table can be found at <Document 1476100.1> - E1: DD: LNGP (Languge, LanguagePreference) and UDCs 01/LP, H79/AS and H95/LN
  • Central Object or specification in EnterpriseOne can be found at <Document 1142313.1> - E1: ENV: Overview of Specifications (SPEC) in EnterpriseOne
  • In P9220 put language to 'CT' and verify that Form Tile is not displayed correctly
  • Central Object F98750 contains FDATEXT information
  • Text ID can be collected through Vocabulary Override (e.g., 2323)
  • P9220 does not allow you to override Form Title so refer to below workaround
  • Return Carriage '\n' reside in between '訂單' and '號碼' which appears to be a space because this has to be the 1st column of grid.

Now the user has verified Text ID and what is problem for this issue. So below steps will guide you how to override it to render form/grid of W4314B correctly. For further detail, refer to <Document 632443.1> - E1: FDA: Translate the Form Title based on Language Preference

Note:

  • (1) Form Title is not written correctly
  • Log on to user specific language (e.g., CT)
  • Go to Form Design Aid
  • Add TV (Text Variable) .Refer (2)
  • Call System Function in General Section of system function "Set Form Title" at the first line of "Dialog Is Initialized" Event
  • Check In
  • Build client package
  • Test it. Refer (3) which shows Form Title correctly and Grid is not collapsed and form title appears correctly

Solution

Refer <Bug 16069933> - W4314B GRID COLLAPSE FOR CT LANGUAGE PROFILE

If issue is specific to a certain set of data, refer to <Document 947798.1> - E1: FDA: Grid Disappears When Using "Go To End" on Web Client

Currently this fix is availble with ESU name "TCH_FIX_006" which you can find from Change Assistant.
