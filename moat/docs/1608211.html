Purpose

Overview

The Pricing Unit of Measure basic preference allows a pricing unit of measure that is different from the pricing unit of measure set up for the item to default into the Pricing Unit of Measure field in the sales order detail record.

The system will search through the preference hierarchy until it finds a match. The pricing unit of measure (UOM4) field in the Sales Order Detail file (F4211) is updated.

Details

Set Up

  1. Preference Definition (Menu G4231)


From Preference Master (P40070), inquire on Pricing Unit of Measure and take Row Exit to Hierarchy (P40073).



Verify that the hierarchy is set up as desired.  For more information on the importance of setting this up, see E1: 45: Preference Hierarchy Functionality for Advanced Pricing (P40073, P4074, P4071, P4072, G4231112) (Doc ID 1346821.1).



To set up the preference, in Preference Master (P40070) take the Row Exit to Profiles, then add the new preference. Alternatively, the preference profiles can be accessed directly in P40302 (Unit of Measure Preference Profiles).


Hierarchy selections – in this example, the preference will be set up by Item / Customer



Enter the preference set up information


2.    Sales Order Entry (Menu G4211)


Sales Order Entry (P4210) Processing Options:

Defaults tab – Option #8 = 1 for Pricing Unit of Measure (optional, but being set as Pricing Unit of Measure for this example).



Preference Tab – activate preference processing



Versions tab – version of Preference Profile (R40400) is the default, which is ZJDE0001



Preference Profile Processing (R40400) – activated Pricing Unit of Measure


3.    Item Set Up (Menu G4111)

Item Master (P4101):
On Weights and Measures tab, Pricing Unit of Measure is set up as EA.



Item Unit of Measure Conversion (P41002) set up as 1BX = 10 EA

4.    Item Price (P4106, Menu G4222)

In this example, the Base Price is set up as $25 EA

Example at Sales Order Entry

Enter a sales order for the item and customer defined in the preference, and the Pricing Unit of Measure is BX, as expected.  Unit Price was correctly calculated as 1 BX = 10 EA, at $25 EA, totaling $250.



Note: In the previous screen shot, the Transaction UOM is BX, which is based on the Default Unit of Measure processing option behind Sales Order Entry (P4210).  If this option were blank, it would display the Primary UOM, which is EA.  The Unit Price and Pricing UOM would be the same.  However, notice that the Extended Price shows the EA price, $25.00.  This is due to the Unit Price displayed being for BX, but the order line is for 1 EA



When this same item is entered on a sales order for a customer not defined in the preference, the Pricing UOM is taken from the Item Master (P4101) definition.  Unit Price on the order will be the EA price defined in Base Price (P4106)

Kit Processing

The Pricing Unit of Measure preference will resolve at sales order entry for kit parent items. Its supported for Kit/Configurator Pricing Methods 1 - Total Component List Prices, 2 – List Price of Final Kit, and 4 - Components Discounted Prices (Advanced Pricing).  


For additional information on setting up and processing kits, see Kit Processing Overview P4210/P42101/P3002/P4004Z/P41351) (Doc ID 625682.1)

Example:

The following set up is required to apply this preference to kit parent items:

Item Master (P4101) – the Kit/Configurator Pricing Method is 1 - Total Components List Prices.  



Bill of Material (P3002, Menu G4114) – in this example, the kit parent and components are set up with EA as their Unit of Measure.



Preference Master (P40302, Menu G4231) – set up the kit parent item to use BX as the Pricing UOM



Item Unit of Measure Conversion (P41002)



Component Base Prices (P4106) – total of the two is $35, which is rolled up as the kit unit price (EA).



At Sales Order Entry (P4210/P42101), the preference is applied, and the Unit Price is correctly calculated based on the roll up of the component prices, then multiplied by 10 for the conversion to BX ($35.00 * 10 = $350.00).



Note:  In this example, the Transaction UOM is displayed in the Pricing UOM based on P4210 processing option, Defaults tab, Option 8.  

Kit Selection Window shows Quantity 10 EA for both of the components (10 EA = 1 BX).

Additional Information

•    Pricing Unit of Measure Preference is a basic preference only, unless a bulk item.
•    For information on the basic and advanced preference for bulk items, see Pricing Unit of Measure (Branch or ECS) Basic and Advanced Preference Profiles (P40347, P4070, P4071, P4072)).
•    Kit Processing at the parent item level is supported with this preference for Kit/Configurator Pricing Methods 1, 2 and 4.
