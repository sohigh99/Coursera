(Troubleshoot) How to call business functions to compute Forward/Backward Schedule, Leadtime and Shrinkage Quantity

Purpose of Document

Purpose for this document is to explain how to call standard EnterpriseOne business functions to meet your daily requirement.

 

In EnterpriseOne Sales/Procurement contain multiple date columns to record accurate information from your business parties. Possibly you may refer below document to meet your daily requirement. Oracle do not advise you to modify any standard functionality so this document is reference only. In case work order is originated from distribution applications, which call below business functions to set valid date for Work Order.

Before you proceed further, refer to below documents:-

<Document 626117.1> E1: 31: P48013 Work Order Start Dates & Back Scheduling Calculations
<Document 881632.1> E1: 31: How Work Order Start Date is Calculated for Variable Lead Time
<Document 660707.1> E1: 34: R3482/R3483 Basics of MRP Dates and Leadtimes

Note:

  • All business function can be called directly from application
  • Before you call B3100300, you can call B3100290 which returns leadtime can be used by B3100300
  • B3100290 is calling B3100250 to calculate quantity however this does not affect the outcome of B3100300


Business Function: B3100300 - ForwardBackwardScheduleDates  (F0007 Calculate Forward Backward Schedule Dates)

  • Purpose: This business function will forward or backward schedule the input date using the Work Day Calendar Table F0007.
  • Parameter:
    • Data Structure: D3100300
Parameter NameData ItemData TypeReqI/OOthers
szBranchPlant MMCU char OPT I Work Order Branch Plant
If there is no definition look for on *ALL branch from F0007
jdInputDate DRQJ JDEDATE OPT I Date to be Validated
jdOutputDate STRT JDEDATE OPT O Scheduled Date
mnNumberOfDays LTLV MATH_NUMERIC OPT I No of Days to forward/backword
cScheduleType WS char OPT I
  • F - Forward Schedule
  • B - Backward Schedule
If you do not specify (that is, Blank or NULL) then defaults 'B'
cErrorCode ERRC char OPT O
  • 1 - if  the Date is invalid.
  • 0 - if the Date is valid work day.
szWorkDayCalendarType WDCT char OPT I  
szWorkDayCalendarKey WDCK char OPT I  

 

  • Detail Routine:
    • Pull out the month and year from the WorkDate and Strip the century off the year.  Read the record from the Work Day Calendar table for the month and year.
    • Read the Work Day Calendar record from F0007 for the input Branch, Month, and Year.  Return the record.
    • select * from proddta.f0007 where (CZMMCU = szBranchPlant and CZWDCT = szWorkDayCalendarType and czwdck = szWorkDayCalendarKey and czctry = centuryofjdInputeDate and czyr = yearofjdeInputDate and czmt = monthofjdInputDate and czshft =''). For example, SELECT  *  FROM PRODDTA.F0007  WHERE  ( CZMMCU = '         M30' AND CZWDCT = ' ' AND CZWDCK = ' ' AND CZCTRY = 1.000000 AND CZYR = 13.000000 AND CZMT = 4.000000 AND CZSHFT = ' ' ) 
    • If data not found then repeat same query based on MMCU = '        *ALL'
  • Example of computation:
    • Test 1: Set jdInputDate [18-04-2013], mnNumberOfDays  [5] and cScheduleType [B]  which back schedule to 5 days based on 18-04-2013. As we can see below, jdOutputDate [11-04-2013]  is to be set.
    • Test 2: Set jdInputDate [26-04-2013], mnNumberOfDays  [7] and cScheduleType [F]  which forward schedule to 7 days based on 26-04-2013. Output jdOutputDate [07-05-2013]

  • Error: No error code is specified.
  • Business function documenation: b3100290.rtf
  • Others: refer B3100290 - CalculateOrderLeadTime

 

Note:

  • This example shows a typical string variable cDayArray[31] which is character array with '\0' (Null character or Terminate character in C programming language).
  • Since table F0007 is a member of F98613 (Database Caching) you must reset database cache if you have changed any.
  • In case you want to view which day it falls in you may implement B0700068 - RetrieveDayofWeek (Retrieve Day of Week) based on value returned.

Business Function: B3100290 - CalculateOrderLeadTime (F3009 Calculate Order Lead Time)

  • Purpose: This business function will calculate the Lead Time required for a given Item, Branch and Order Quantity.
  • Parameter:
    • Data Structure: D3100290
Parameter NameData ItemData TypeReqI/OOthers
szBranch MCU char OPT I Detail Branch or Work Order Branch
mnShortItem ITM MATH_NUMERIC OPT I Item Number - Short
mnOrderQuantity UORG MATH_NUMERIC OPT I Quantity to be used in computing quantity with Shrinkage
szUnitOfMeasure UOM char OPT I Quantity to be used in computing shrinkage
mnLeadtimeDays LTLV MATH_NUMERIC OPT O Computed leadtime value after shrinkage computation
cErrorCode ERRC char OPT O Issue Warning
cBackscheduleQueueMoveHrsFlag EV01 char OPT I 1 - To backschedule queue hours
or 0
  • Detail Routine:

        

        LeadTimeInHours = LeadTimeInHours (= LeadTimeHours / TimeBasisRatio) + StandardSetupHours + QueueHours
        LeadTimeDays = LeadTime In Hours / Time Basis Code

  • Error: No Error Code is implemented
  • Business function documenation: b3100290.rtf

F4102 LTCV LeadtimeConversionFactor (Leadtime Conversion Factor) A number that the system uses to convert leadtime per unit hours to leadtime per unit days. The leadtime conversion factor is stored in the Item Branch File table (F4102). The system uses leadtime per unit days, cumulative move days, and setup days to calculate the start date for backscheduled work orders. The leadtime conversion factor is based on the sum of the labor hours for each work center shift. The leadtime conversion factor is invoked when the system runs the Leadtime Rollup program (R30822A).
Note: The leadtime conversion factor is used only if you specify a variable leadtime in the Item/Branch record. If you specify a fixed leadtime, the system does not use the leadtime conversion factor to calculate leadtime.		 JDB_SelectKeyed()

F4102 LTLV LeadtimeLevel (Leadtime Level) A value that represents the leadtime for an item at its assigned level in the production process, as defined on Plant Manufacturing Data. The system uses this value to calculate the start dates for work orders using fixed leadtimes.
Level leadtime is different for purchased and manufactured items:
Purchased - The number of calendar days required for the item to arrive at your branch/plant after the supplier receives your purchase order.
Manufactured - The number of workdays required to complete the fabrication or assembly of an item after all the components are available.
You can enter level leadtime manually on Manufacturing Values Entry, or you can use the Leadtime Rollup program to calculate it. To calculate level leadtime using the Leadtime Rollup program, you must first enter a quantity in the Manufacturing Leadtime Quantity field in the Item Branch table (F4102).		 GetITemBranchMfgData
