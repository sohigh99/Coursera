Archive and Purging within JD Edwards EnterpriseOne

Goal
Solution

Goal

This document describes how to perform Archive and Purging within EnterpriseOne.

Solution

In reference to archiving, there are no specific tools available within EnterpriseOne. You can use any third party tools and features available within your Database (RDBMS native tools).

Note: Some UBEs in Distribution and Financial Modules have functionality to archive data before it is purged/deleted.

In reference to purging, EnterpriseOne provides some UBES to purge various tables. These tables belong to system as well as functional (business data).

Here are few purge UBES that purge records from system tables:

UBEDESCRIPTION / TABLE(S)
R01131P Purge work center messages from: F01131, F01131T,  F01131M, F01133, F00165 and F00166
R91300B             Scheduler Purge Program
R9312 Purge Security History from F9312
R95921 Purge Role Relationship
R959212 Purge Obsolete Role Relationships
R96225 Package History Purge
R9647PX Where X is 2-9, 11, 12 -- Delete respective objects from F9860
R9650 Machine Master Purge
R967002 Delete Files and Folders from Directory
R980011B Purge Cross Reference Records
R98210B Object Management Log Purge
R98222B Purge OMW Project
R986101B OCM Add/Update/Delete
R9861101 Purge Job Master from F986110
R986120 Object Configuration Delete

 

Some Application Specific Purge UBE:

UBEDESCRIPTION
R03B312 Customer Activity Log Purge
R03B673 Purge Draft Remittance Records


For additional information, follow the attached whitepaper. The document "Processes to Purge Data for JD Edwards EnterpriseOne" contains details of the purge UBE available within EnterpriseOne.

NOTE:  R4101P is to be running after you run R4102P successfully as R4102P has detail routine to check whether there is any open transaction across EnterpriseOne. So R4102P validates transactional file based on master file whereas R4101P validates against master files. R4102P will delete records from F4105 and F4102 based on the PO Values (Option 3#) of R4102P.
