Purpose

Overview

Advanced Preferences have the ability to default customer-preferred carrier/mode of transport/services as specified in the customer profile.

Beginning with release 8.10, all the basic transportation preferences are available as advanced preferences. The system will resolve advanced preferences first then look for standard preferences. Transportation Management supports the following advance preference types:

Transportation Advanced Preferences
Type Description Advanced Preference Functionality
12 Document Set Transportation Management
13 Options and Equipment Transportation Management
14 Customer Freight Transportation Management
15 Carrier Transportation Management
16 Mode of Transport Transportation Management

The transportation preferences are resolved when the shipment is created, in the sequence listed:

  1. Customer Freight Preference
  2. Mode of Transport Preference
  3. Carrier Preference
  4. Options and Equipment Preference

The Document Set Preference is resolved whenever shipment documents are printed and the Document Selection (P49591) is called.

Select the System Constants form exit from Branch/Plant Constants (P41001) and populate the Preference Schedule field with the advanced preference schedule. For transportation it is mandatory to have the schedule in System Constants. Transportation will not consider the advanced preference schedule associated with the P4210 processing options.

Customer Freight Preference: Advance Preference Type 14

This preference provides several defaults (or return values):

  • Freight Rate Schedule
  • Freight Handling Code
  • Route Selection Type
  • Billable Freight Summary/Detail flag
  • Distance
  • Distance UOM

Customer Freight Preference: Advanced Preferences Name Revisions (P4071)

From menu G40311 select Advanced Preferences Name Revisions (P4071).
Create a new customer freight preference: click Add and specify:

  • Preference name: for example TAFR. Note: Creating a new Preference Name will generate an entry in the 40/TY UDC table. A '2' will be placed in the Special Handling Code to indicate that this is a Preference Name rather than a Price Adjustment.
  • Item Group / Customer Group: optional (needed when complex groups are used). Note: If complex groups are used, the customer /item group needs to be specified here in the preference name.
  • Preference Hierarchy: required.
  • Advance Preference: 14 (Customer Freight).
  • Level break type: None (or Quantity level break).



Customer Freight Preference: Advance Preference Detail Revision (P4072)

From menu G40311 select Advanced Preference Detail Revisions (P4072).

On Advance Preference Detail, search for the Preference Name defined as the previous step in P4071 and click Add to create a new preference detail.

On Preference Hierarchy Selection, select the appropriate combination and click Select.

      Select the line and take Pref Details form exit.



      In P4072 (W4072D form) enter the additional Driver fields (like Mode of Transport) and the Return fields. For the Options and Equipment preference, the return fields are:

      • Delivery Line Level: this field is used to indicate that a freight charge is calculated once per detail line, once per delivery, or once per load/trip.
      • Options/Equipment: option or piece of equipment which is associated with a shipment or which is required in order to make a shipment.

      This preference can be used in a variety of ways. In the simplest case, it can be used for an option that is always required for an item or customer, such as a refrigerated trailer or inside delivery. An option preference is useful (when the requirement is ongoing) so that the person taking the order does not have to manually attach (or even know about) the option. This preference can also be used to drive selection (or suppression) of certain carriers or modes for special requirements. This is because once an option is attached to a shipment; the option must be supported by a route, carrier and mode in order for a specific routing option to be selected. For example, certain items cannot be shipped by air. In this case, air-based routes can be suppressed by adding a default option/equipment code that is incompatible with these routes.

Frequently Asked Questions

Question 1: How to activate advanced transportation preferences?

Answer 1: To activate the advanced transportation preferences setup the Preference Schedule in System Constants (P41001) in order to be resolved. While the transportation base preferences are "always on", the advanced ones need to have the schedule set up in Branch/Plant constants in order to be resolved. Transportation preferences will not be activated by setting up the processing options on the Preference tab of Sales Order Entry (P4210). The schedule set up there overrides the one in Branch Plant Constants only when resolving sales advanced preferences.

Question 2: In which sequence the transportations preferences are resolved?

Answer 2: Transportation preferences are resolved at the time a shipment is created, in the sequence listed:

      1. Customer Freight Preference
      2. Mode of Transport Preference
      3. Carrier Preference
      4. Options and Equipment Preference

The Document Set Preference is resolved whenever shipment documents are printed and the Document Selection (P49591) is called.

Question 3: What happens when there are override values?

Answer 3: Transportation preference values will never override values that are manually keyed in a transaction. The values will be based on the following sequence:

      1. Manually keyed
      2. Non-blank values in Customer Master Table
      3. Preference profile value


Question 4: What are the driver fields and the return fields when using advance preference?

Answer 4: The driver and return fields are the key fields that will appear on P4072 (form W4072D) dependent upon the Advanced Preference Type.

The driver fields are additional fields used to select a preference. The return values are the values default or the values resulted after applying the preference. Each preference type has a different set of return values. A return value can be a driver value for a subsequent preference. For example, when a Mode of Transport preference is resolved, the mode will be used as a driver for subsequent preferences that have Mode of Transport (MOT) for the driver field (Carrier Preference and Options and Equipment Preference).
